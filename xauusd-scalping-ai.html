⚠️ If Access Logs Are Empty — Run This SQL in Supabase

The EA inserts rows via the anon key. Without RLS INSERT permission, all log writes silently fail. Run these in your Supabase SQL Editor:

-- 1. Allow anon to insert access logs (EA uses anon key) ALTER TABLE access_logs ENABLE ROW LEVEL SECURITY; DROP POLICY IF EXISTS "allow_anon_insert_logs" ON access_logs; CREATE POLICY "allow_anon_insert_logs" ON access_logs FOR INSERT TO anon WITH CHECK (true); -- 2. Allow anon to read licenses (for EA validation) DROP POLICY IF EXISTS "allow_anon_select_licenses" ON licenses; CREATE POLICY "allow_anon_select_licenses" ON licenses FOR SELECT TO anon USING (true); -- 3. Allow anon to update licenses (last_seen, account_number) DROP POLICY IF EXISTS "allow_anon_update_licenses" ON licenses; CREATE POLICY "allow_anon_update_licenses" ON licenses FOR UPDATE TO anon USING (true);

Also confirm your access_logs table has these columns: id (uuid default), license_key (text), account_number (text), ip_address (text), terminal_name (text), login_time (timestamptz default now()). If columns are missing, add them:

ALTER TABLE access_logs ADD COLUMN IF NOT EXISTS terminal_name text; ALTER TABLE access_logs ADD COLUMN IF NOT EXISTS ip_address text;
